# Structure de base d’une application

## Objectif
Comprendre comment une application moderne est organisée en deux parties principales : le backend (MonApi) et le frontend (MonFrontend).

## Explications détaillées

- **Projet** : Un ensemble de fichiers et de dossiers qui servent à développer une partie d’une application.
- **MonApi** :
  - Projet backend (arrière-plan)
  - Gère la logique, les données, les échanges avec la base de données
  - Appelé aussi « API »
- **MonFrontend** :
  - Projet frontend (interface utilisateur)
  - Affiche ce que voit l’utilisateur, gère les interactions (boutons, formulaires…)
  - Réalisé ici avec React

## Points à retenir

- Une application moderne est souvent divisée en deux projets : backend (API) et frontend (interface).
- Chaque projet a son propre dossier et ses propres fichiers.
- Le backend s’occupe des données et de la logique ; le frontend s’occupe de l’affichage et des interactions.

## Conseils pratiques

- Toujours bien séparer le code backend et frontend pour une meilleure organisation.
- Utiliser les dossiers pour ranger les fichiers selon leur rôle.
